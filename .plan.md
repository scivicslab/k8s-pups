# ホストファイルシステムアクセス実装計画

## 概要
k8s-pupsログインユーザーの$HOME（NFS: `/Public/Users/{username}`）をPodにマウントする。
LDAP(`ldap-set2.sc-account-bg.svc:389`)からuidNumber/gidNumber/homeDirectoryを取得し、
ユーザーがPOSIXアカウントを持たない場合はサイレントスキップ。

## 対象プラグイン
| プラグイン | マウント先（Pod内） | 用途 |
|-----------|-------------------|------|
| Remote Desktop | `/home/{username}` | ユーザーの$HOMEとしてデスクトップ環境 |
| Jupyter Lab | `/home/jovyan` | ユーザーのnotebook/ファイルへアクセス |
| Docusaurus | `/workspace` | ~/works以下のDocusaurusプロジェクト |

## 実装ステップ

### Step 1: devteamをsc-account-bg LDAPに追加
- `ou=admins,ou=accounts,dc=nig,dc=ac,dc=jp`にdevteamエントリ作成
- uidNumber=1002, gidNumber=1002, homeDirectory=/home/devteam

### Step 2: LdapUserInfoクラス新規作成
- LDAP接続してuidNumber/gidNumber/homeDirectoryを取得
- ユーザー未登録 or posixAccount属性なし → null返却（サイレントスキップの根拠）
- 接続情報はapplication.propertiesで設定
- LDAP接続パスワードはVaultから取得

### Step 3: ToolPluginインターフェース変更
- `workspaceEnabled()` 追加（default false）— NFS $HOMEマウントを希望するか
- 既存の`workspacePvcName()`/`workspaceMountPath()`は削除（Docusaurus固有だった）

### Step 4: K8sApiClient変更
- `createWorkspacePvPvcIfAbsent(userId, nfsServer, nfsShare)` メソッド追加
  - PV: `pups-workspace-{userId}` → NFS `{nfsServer}:/{nfsShare}`
  - PVC: `pups-workspace-{userId}` (namespace: user-pods)
  - 既に存在すれば何もしない
- `buildPodSpec()`変更:
  - workspaceが有効でPVCが用意された場合 → マウント
  - `runAsUser`/`runAsGroup`をLDAPのuidNumber/gidNumberで上書き

### Step 5: SessionInfo/SessionActor変更
- SessionInfoにworkspace情報を追加（userUid, userGid, workspacePvcName）
- SessionActor.start()で:
  1. plugin.workspaceEnabled() == true の場合
  2. LDAPクエリでユーザーのPOSIX情報取得
  3. 取得成功 → PV/PVC作成 → SessionInfoにセット
  4. 取得失敗 → スキップ（既存のuserDataMountPath動作にフォールバック）

### Step 6: 各プラグイン変更
- **GuacamolePlugin**: workspaceEnabled()=true
  - NFS有り: $HOME at `/home/{username}`, runAsUser=LDAP UID
  - NFS無し: 従来通りPVC at `/home/user`, runAsUser=1000
- **JupyterLabPlugin**: workspaceEnabled()=true
  - NFS有り: $HOME at `/home/jovyan`, runAsUser=LDAP UID
  - NFS無し: 従来通り
- **DocusaurusPlugin**: workspaceEnabled()=true
  - NFS有り: ~/works at `/workspace`, runAsUser=LDAP UID
  - NFS無し: エラー（Docusaurusはworkspace必須）

### 設定項目（application.properties）
```
k8s-pups.workspace.nfs-server=192.168.5.20
k8s-pups.workspace.nfs-base-path=/Public/Users
k8s-pups.ldap.url=ldap://ldap-set2.sc-account-bg.svc:389
k8s-pups.ldap.base-dn=dc=nig,dc=ac,dc=jp
k8s-pups.ldap.bind-dn=cn=admin,dc=nig,dc=ac,dc=jp
# bind password from Vault
```
